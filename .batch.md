# Batch: Extract Payload.luau from Worker.luau (Round 1)

## Task 1: Extract Payload.luau [tier: standard]
Refactor `src/parallel/Worker.luau` in this Lune/Luau project (strict types, `--!strict`
at top of every module — see CLAUDE.md for project conventions). Today `Worker.luau` is a
standalone Lune script that reads a payload from stdin, does pure buffer-in/buffer-out
encode or decode logic inline, and writes the result to stdout. It has two entry points
selected by CLI arg: `runEncode()` and `runDecode()`.

Goal: pull the **pure, transport-agnostic** logic (everything that doesn't call
`stdio.readToEnd()` / `stdio.write()` / `process.exit()`) out of `runEncode()` and
`runDecode()` into a new module `src/parallel/Payload.luau`, so a future second transport
(not part of this task) can reuse the exact same logic without going through stdin/stdout.

### What to do

1. Create `src/parallel/Payload.luau` with `--!strict` at the top. Export two functions
   with this exact shape:

   ```luau
   export type Payload = {
       encode: (input: buffer) -> buffer,
       decode: (input: buffer) -> buffer,
   }

   local Payload = {}

   function Payload.encode(input: buffer): buffer
       -- body of today's runEncode(), minus the stdio.readToEnd()/stdio.write() calls:
       -- takes the raw input buffer (what stdio.readToEnd() used to hand to
       -- buffer.fromstring), returns the raw output buffer (what used to be passed to
       -- buffer.tostring(...) before stdio.write()).
   end

   function Payload.decode(input: buffer): buffer
       -- same idea for today's runDecode()
   end

   return Payload
   ```

   Move ALL the logic verbatim — the `BufferReader`/`BufferWriter` construction, the
   synthetic `ArtifactData` building, the `ColumnEncoder`/`V4.readClassBlockBody` calls,
   the bitmask packing/unpacking, everything currently inside `runEncode`/`runDecode` in
   `src/parallel/Worker.luau` except the stdio calls themselves. Do not change any byte
   of the wire format or any behavior — this is a pure move/refactor, not a rewrite.
   Preserve all existing comments that explain wire format (they're load-bearing
   documentation of the protocol) — move them to `Payload.luau` alongside the code they
   describe.

2. Update `src/parallel/Worker.luau` to become a thin wrapper:
   - `require("@src/parallel/Payload")` (this project uses `@src/...` require aliases,
     see `.luaurc`)
   - `runEncode()` becomes: read stdin via `stdio.readToEnd()` → `buffer.fromstring` →
     `Payload.encode(buf)` → `buffer.tostring` → `stdio.write(...)`
   - `runDecode()` becomes the same shape calling `Payload.decode`
   - Keep the CLI arg parsing (`process.args` scan for `"decode"`) and the top-level
     `pcall` + `stdio.ewrite` + `process.exit(1)` error handling exactly as-is — that
     stays in `Worker.luau`, it's transport-level, not pure logic.
   - Keep the big top-of-file comment block describing the wire protocol in
     `Worker.luau` (or move/duplicate the relevant parts to `Payload.luau` — your call,
     but don't delete it; it's the only documentation of this protocol).

3. Do NOT change `src/instance/Serializer.luau` or `src/instance/Deserializer.luau` —
   they invoke `Worker.luau` as an external process via `process.exec`/`process.create`
   and don't need to know about this internal refactor.
